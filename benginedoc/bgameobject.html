Detailed Description

The BGameObject class is the base class for all Game Objects.

A BGameObject carries a list of BGameComponent objects and is the base class for all game objects.

See also BGameComponent.

Member Function Documentation

Constructs a new BGameObject.

Destroys a BGameObject.

void BGameObject::addChild(BGameObject * child)

Designates child as a child of this game object.

If child is already a child of this or any other game object, it will not be added as a child to this.

void BGameObject::addComponent(BGameComponent * component)

Adds component to this object's list of components.

If component already belongs to this object, it will not be added again. Also, if component already belongs to another BGameObject, it will not be added to this. If the component is added, it automatically sets the owner of component to this.

Note: A BGameObject has ownership of its BGameComponent objects and is responsible for deleting them.

bool BGameObject::removeChild(BGameObject * child)

Designates that child should no longer be a child of this game object.

Returns true if child was initially a child of this game object and if it was removed, false otherwise.

bool BGameObject::removeComponent(BGameComponent * component)

Removes a BGameComponent from the object.

Automatically sets the parent of component to null if removed.

Returns true if component was owned by this BGameObject and removed; false otherwise.

Note: If you remove a BGameComponent from a BGameObject, it is your responsibility to free its memory to avoid a memory leak.

void BGameObject::setParent(BGameObject * newParent)

Sets the parent BGameObject of this game object to newParent.

If this object was previously the child of another game object, it will notify the other game object that it's about to peace out.

See also hasParent().
