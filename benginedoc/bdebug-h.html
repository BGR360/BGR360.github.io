<bdebug.h> - Debug Logging Macros

The <bdebug.h> header file provides useful macros for generating debug and error messages. More...

Macros

B_CRITICAL( message)
B_DEBUG( message)
B_FATAL( message)
B_INFO( message)
B_WARN( message)

Use these handy macros to do all of your debugging and error reporting.

Macro Documentation

B_CRITICAL( message)

Prints a critical error message or a system error message.

Along with B_WARN, you can use this macro to report recoverable errors in your code. Use it rarely, for errors that are more serious than warnings but are still recoverable. This macro will not terminate the program.

The message parameter can be either a c-string, QString, number, or pretty much anything that can be inserted using << into the qDebug() stream.

B_DEBUG( message)

Prints a debug message.

The message parameter can be either a c-string, QString, number, or pretty much anything that can be inserted using << into the qDebug() stream.

B_FATAL( message)

Prints a fatal error message shortly before terminating the program.

Only use this macro for truly fatal errors.

The message parameter can be either a c-string, QString, number, or pretty much anything that can be inserted using << into the qDebug() stream.

B_INFO( message)

Prints an informational message.

The message parameter can be either a c-string, QString, number, or pretty much anything that can be inserted using << into the qDebug() stream.

B_WARN( message)

Prints a warning message.

You should primarily use this macro for any errors that might be encountered in your code. This macro does not terminate the program, so only use it for recoverable/non-fatal errors.

The message parameter can be either a c-string, QString, number, or pretty much anything that can be inserted using << into the qDebug() stream.
